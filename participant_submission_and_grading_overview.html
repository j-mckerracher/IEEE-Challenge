Participant Submissions and Automated Grading

For the IEEE Computer Society Global Student Challenge: FRESCO early job failure prediction.

Submission Package

Each team should submit three required artifacts. The prediction file is used for automated scoring, while the code and report support the methodology, creativity, efficiency, and documentation review.

1. Prediction CSV

The CSV must contain one row for each test example and exactly one predicted failure probability per row.

  • row_id: identifier copied from the test set
  • failure_probability: numeric probability from 0 to 1

Higher values should indicate greater predicted risk that the job will fail or time out.

2. Reproducible Code or Notebook

Teams should provide a Python script, package, or notebook that shows how the predictions were produced. It should include preprocessing, feature construction, model training, validation, and final CSV export.

3. Short Methodology Report

The report should explain the modeling choices and validation process. Strong reports should discuss feature design, model choice, leakage controls, class imbalance, cluster-aware validation, cluster differences, and any creative or advanced techniques used.

Recommended Rubric

  • Prediction Accuracy (70%): automated score based on the hidden test labels, primarily macro-average AUPRC across clusters.
  • Methodology and Creativity (20%): appropriateness, originality, and rigor of the approach, including leakage-aware validation and cluster-aware modeling.
  • Efficiency and Documentation (10%): clean, modular, reproducible, and computationally reasonable code with clear explanations.

Automated Grading Script

The automated grader should validate the submission format, join predictions to hidden labels by row_id, compute per-cluster AUPRC, and return one numeric score. Macro-average AUPRC is recommended because it prevents the largest cluster from dominating the leaderboard.

The production-style Python scorer for Kaggle is provided in kaggle_score_submission.py. The example below shows the same scoring idea in a compact form.

Expected grader inputs
  • submission.csv: participant predictions with row_id and failure_probability
  • hidden_labels.csv: organizer-held labels with row_id, cluster, and label
from __future__ import annotations

import argparse
import json

import numpy as np
import pandas as pd
from sklearn.metrics import average_precision_score, roc_auc_score


def validate_submission(submission: pd.DataFrame, labels: pd.DataFrame) -> None:
    required_columns = {"row_id", "failure_probability"}
    missing = required_columns - set(submission.columns)
    if missing:
        raise ValueError(f"Submission is missing required columns: {sorted(missing)}")

    if submission["row_id"].duplicated().any():
        raise ValueError("Submission contains duplicate row_id values.")

    expected_ids = set(labels["row_id"])
    submitted_ids = set(submission["row_id"])
    missing_ids = expected_ids - submitted_ids
    extra_ids = submitted_ids - expected_ids
    if missing_ids or extra_ids:
        raise ValueError(
            f"Submission row_id mismatch: {len(missing_ids)} missing, "
            f"{len(extra_ids)} unexpected."
        )

    probabilities = submission["failure_probability"]
    if not np.isfinite(probabilities).all():
        raise ValueError("failure_probability must contain only finite numeric values.")
    if ((probabilities < 0) | (probabilities > 1)).any():
        raise ValueError("failure_probability must be between 0 and 1.")


def score_submission(submission_path: str, labels_path: str) -> dict:
    submission = pd.read_csv(submission_path)
    labels = pd.read_csv(labels_path)

    validate_submission(submission, labels)

    merged = labels.merge(submission, on="row_id", how="left", validate="one_to_one")
    per_cluster = {}

    for cluster, group in merged.groupby("cluster"):
        y_true = group["label"].astype(int)
        y_score = group["failure_probability"].astype(float)

        # AUPRC is undefined when a cluster has only one class in the hidden test set.
        if y_true.nunique() < 2:
            continue

        per_cluster[str(cluster)] = {
            "auprc": float(average_precision_score(y_true, y_score)),
            "auroc": float(roc_auc_score(y_true, y_score)),
            "n_rows": int(len(group)),
            "positive_rate": float(y_true.mean()),
        }

    if not per_cluster:
        raise ValueError("No cluster had both positive and negative examples.")

    macro_auprc = float(np.mean([metrics["auprc"] for metrics in per_cluster.values()]))

    return {
        "score": macro_auprc,
        "primary_metric": "macro_cluster_auprc",
        "per_cluster": per_cluster,
    }


def main() -> None:
    parser = argparse.ArgumentParser()
    parser.add_argument("--submission", required=True)
    parser.add_argument("--labels", required=True)
    args = parser.parse_args()

    result = score_submission(args.submission, args.labels)
    print(json.dumps(result, indent=2, sort_keys=True))


if __name__ == "__main__":
    main()

Notes for Organizers

  • Keep hidden_labels.csv private.
  • Use a temporal split within each cluster to reduce leakage from future jobs.
  • Provide a sample_submission.csv so teams can match the required format.
  • Consider publishing per-cluster public validation scores, but reserve hidden test labels for final ranking.
